## 커밋 규칙 

| 타입     | 설명                                                           |
| -------- | -------------------------------------------------------------- |
| Feat     | 새로운 기능 추가                                              |
| Chore    | 빌드 업무 수정, 패키지 매니저 수정                             |
| Fix      | 버그 수정                                                      |
| Style    | 스타일 관련 기능(코드 포맷팅, 세미콜론 누락, 코드 자체의 변경이 없는 경우)       |
| Refactor | 코드 리팩토링                                                  |
| Design   | css등 UI 변경                                                  |
| Comment  | 주석 추가, 변경                                                |
| Docs     | 문서 수정                                         |
| Test     | 테스트 코드 추가                                                    |
| Rename   | 파일 or 폴더명 수정하거나 옮기는 작업만 수행할 경우            |
| Remove   | 파일 삭제만 수행                                               |
